org-super-agenda                                              *org-super-agenda*

This plugin organizes your orgmode agenda in a floating window using
custom groups. It is inspired by the Emacs plugin org-super-agenda.

===============================================================================
USAGE                                                           *org-super-agenda-usage*

```
:OrgSuperAgenda
    Opens the agenda window (floating). Use <CR> on a headline to jump to
    the source location. Use `q` or <Esc> to close the window.

:OrgSuperAgenda!  or  :OrgSuperAgenda fullscreen
    Opens the agenda **fullscreen**. Border style is configurable via
    |org-super-agenda-options| (`window.fullscreen_border`).
```

Navigation & actions \~
Press the keymap defined for a TODO state to filter by it and
`oa` to reset the filter. Use `cs` to reschedule, `cd` to set a
deadline, `t` to cycle TODO state, `s` to set state directly,
and `r` to refresh the agenda from disk. Press `K` to preview
the current headline and `q` to close the preview. Press `<CR>`
to edit a headline in the float, `gf` to go to the selected headline —
closes the agenda and opens the file at the headline position in the
previous window. Press <Tab> on a group header
to toggle collapse/expand; on an item line it runs `fold_item_action`
(default: preview).

```
    Additional useful keys:
        of / oz   Live filters (exact / fuzzy)
        oq        Advanced query prompt (see |org-super-agenda-query|)
        st/sp/sw/sd  Set state directly (TODO/PROGRESS/WAITING/DONE)
        s0        Clear TODO state
        s (wait)  Show state menu with colors
        I / O     Clock in / clock out
        X         Cancel active clock
        gI        Jump to active clock task
        D         Toggle allowing duplicates across groups
        zM / zR   Fold all / unfold all groups
        ov        Switch view mode (classic ↔ compact)
        V         Open custom view picker
        x / gX    Hide current item / reset hidden list
        R         Refile via Telescope (see |org-super-agenda-refile|)
```

===============================================================================
CONFIGURATION                                                   *org-super-agenda-setup*

Call |lua| to setup the plugin:

>

```
lua << EOF
```

```
require('org-super-agenda').setup({
  org_directories     = {},        -- recurse for *.org files
  exclude_files       = {},        -- files to ignore
  exclude_directories = {},        -- directories to ignore
  -- additional options are documented below and in the source
})
EOF
```

<

The `groups` option controls how headlines are grouped. By default items are
grouped into "Today", "Important", "Overdue", "Upcoming" and "Deadlines"
sections. You can define your own groups using Lua functions.

Example groups \~

>

```
groups = {
```

```
  {
    name = "🔥 Urgent Work",
    matcher = function(item)
      return item:has_tag("work") and item.priority == "A"
    end,
  },
  {
    name = "🧪 Experimental",
    matcher = function(item)
      return item.headline:match("Proof of Concept") or item:has_tag("test")
    end,
  },
}
```

<

Matchers have access to:

* `item.todo_state`, `item.headline`, `item.priority`
* `item:has_tag("...")`
* `item.scheduled`, `item.deadline` with methods like `:is_today()` and
  `:days_from_today()`

Groups are checked in order — the **first match wins** unless
|allow\_duplicates| is set.

===============================================================================
SORTING                                                      *org-super-agenda-sorting*

Items within each section can be sorted.

Global fallback \~

>

```
group_sort = { by = 'date_nearest', order = 'asc' }
```

<

Per-group override \~

>

```
{ name = '☠️ Deadlines', matcher = is_deadline,
```

```
  sort = { by = 'deadline', order = 'asc' } }
```

<

Supported sort keys \~
date\_nearest   min(days-to-deadline, days-to-scheduled); missing → ∞
deadline       days from today (negative = past)
scheduled      days from today (negative = past)
deadline_time  exact date+time (includes hour:minute when available)
scheduled_time exact date+time (includes hour:minute when available)
priority       A > B > C
todo           order of your `todo_states` (or custom `sort.todo_order`)
filename       basename (ascending)
headline       case-insensitive title

Tie-breakers: priority desc, then filename asc, then headline asc.

NOTE: Items with time-of-day (e.g., SCHEDULED: <2026-01-31 09:00>) are
displayed with their times in classic view. Use `scheduled_time` or
`deadline_time` sort keys to order by exact timestamp instead of just date.

Custom TODO state order per group \

>

```
{
  name = '📅 Today',
  matcher = function(i) return i.scheduled and i.scheduled:is_today() end,
  sort = {
    by = 'todo',
    order = 'asc',
    todo_order = { 'PROGRESS', 'NEXT', 'TODO', 'WAIT', 'CANX', 'DONE' },
  }
}
```

<

===============================================================================
CLOCKING                                                  *org-super-agenda-clocking*

Clock actions are integrated directly in agenda view:

* `I`    clock in item under cursor
* `O`    clock out current active clock
* `X`    cancel current active clock
* `gI`   jump to active/recent clocked task

Visual indicators:

* active clock items are prefixed with `⏱`
* active clock statusline is shown at the bottom of the agenda window

NOTE: default `reset_hidden` key changed from `X` to `gX` because `X`
is now used for clock cancel.

==============================================================================
GROUP FOLDING                                            *org-super-agenda-folding*

Group headers show item counts in this format:

>

```
* 📅 Today (12 items)
```

<

Use `<Tab>` on a group header to collapse/expand that group.
Use `zM` to fold all groups and `zR` to unfold all groups.

When cursor is on a normal item line, `<Tab>` runs the action configured
via `fold_item_action`:

    'preview'   Show item in preview popup (default)
    'goto'      Close agenda and open file at headline in previous
                window

`gf` always goes to the headline in the previous window, regardless
of `fold_item_action`. To match org-agenda `<Tab>` behavior, set
`fold_item_action = 'goto'`.
Note: `goto` is not available in |org-super-agenda-popup| mode.

==============================================================================
FILTERS & QUERY                                               *org-super-agenda-query*

Quick filters \~
of            Live filter by exact text (type to narrow, <Esc>/<CR> to stop)
oz            Live filter by fuzzy text
oa            Reset filters
ot/op/ow/od   Show only TODO/PROGRESS/WAITING/DONE (configurable)

Advanced query (`oq`) tokens \~
tag\:work|home       require any of these tags
-tag\:chore           exclude tag
file\:notes          filename contains substring
-file\:archive        filename must NOT contain substring
todo\:TODO|WAITING   restrict TODO states
prio>A              compare priorities (A highest); also: prio\:A
due<3               deadline in < 3 days (relative)
sched>=0            scheduled today or later
before:2025-12-31   deadline before absolute date
after:2025-08-01    scheduled after absolute date
is\:overdue          deadline or schedule in the past
is\:done             only DONE
has\:todo            items with TODO state (excludes plain events)
-has\:todo           items without TODO state (events only)
words here          free-text includes (AND)
-noise               free-text excludes

Examples \~

>

```
prio:A tag:work due<5
```

```
tag:personal -WAITING sched>=0
has:todo tag:work
-has:todo tag:personal
```

<

===============================================================================
BULK ACTIONS                                               *org-super-agenda-bulk*

Mark multiple items and apply an operation to all of them at once.

Keymaps ~
m             Toggle mark on current item (● indicator appears)
M             Clear all marks
gv            Reselect last marks (restores marks after clear/bulk action)
B             Open bulk action menu for marked items

Bulk action menu (after pressing B) ~
s             Set TODO state on all marked items
r             Reschedule all marked items (datepicker once, applied to all)
d             Set deadline on all marked items (datepicker once, applied to all)

All bulk operations push individual undo entries onto the undo stack.
Use `u` to undo them one at a time.

Keymaps are configurable via `keymaps.bulk_mark`, `keymaps.bulk_unmark_all`,
`keymaps.bulk_reselect`, and `keymaps.bulk_action`.

===============================================================================
REFILE                                                       *org-super-agenda-refile*

Press `R` on an item to start refiling via Telescope + org-telescope.

* Toggle between **file** and **headline** targets with `<C-Space>`
* The moved segment is written to disk and any unmodified open buffers reload

===============================================================================
DIRECT STATE SETTING                                      *org-super-agenda-set-state*

Set TODO states directly without cycling through all states.

Default keymaps (prefix `s`) \~
st            Set TODO
sp            Set PROGRESS
sw            Set WAITING
sd            Set DONE
s0            Clear state (remove TODO keyword)
s (wait)      Show colored menu

The shortcut key for each state is derived from the first letter of the state
name. If you have conflicting names (e.g., TODO and TRACKING), add an explicit
`shortcut` field to your `todo_states` config:

>

```
todo_states = {
  { name='TODO',     shortcut='t', color='#FF5555', ... },
  { name='TRACKING', shortcut='r', color='#AAAAAA', ... },
}
```

<

When you press `s` and wait (based on your `timeoutlen`), a colored menu
appears showing all available states in their configured colors.

===============================================================================
CUSTOM VIEWS                                             *org-super-agenda-views*

Define reusable, named agenda views with pre-configured filters, groups, sort,
and keymaps. Switch between them via commands, global keymaps, or the in-agenda
picker.

Configuration ~

>

```
custom_views = {
  work_week = {
    name = "Work This Week",
    keymap = "<leader>ow",
    filter = "tag:work sched>=0 sched<7 -is:done",
    groups = { ... },
    sort = { by = 'deadline', order = 'asc' },
    title = "Work Week",
    view_mode = "compact",
  },
  personal = {
    name = "Personal",
    keymap = "<leader>op",
    filter = "tag:personal -is:done",
  },
}
```

<

View options ~
name            Display name for the view (defaults to key name)
keymap          Global keymap to open this view directly
filter          Query string (same syntax as |org-super-agenda-query|)
groups          Override default group structure for this view
sort            View-level sort fallback (between per-group and global)
title           Custom window title override
view_mode       Layout override ('classic' or 'compact')

All fields are optional. A view with just a `filter` is valid.

Accessing views ~
- Global keymaps: defined via `keymap` field, works in normal mode
- `:OrgSuperAgenda view <name>` opens a specific view
- `:OrgSuperAgenda views` opens the view picker
- Press `V` inside the agenda to open the picker

View picker ~
- Numbered selection (press 1, 2, etc. or use <CR>)
- Shows active view with `◀` indicator
- When a view is active, first option is "Default Agenda" to go back
- Press `q` or `<Esc>` to close picker without switching

Sort precedence ~
When a custom view is active, sort is resolved in this order:
1. Per-group `sort` (highest priority)
2. View-level `sort` (from custom view config)
3. Global `group_sort` (lowest priority fallback)

Visual indicators ~
- Window title updates to reflect the active view name
- Footer shows a badge with the active view name and filter string

POPUP MODE                                                  *org-super-agenda-popup*

When running OrgSuperAgenda in a dedicated tmux session (e.g., as a persistent
popup), you can enable `popup_mode` so that pressing `q` hides the session
instead of closing the buffer. This allows instant access to your agenda.

Example setup \~

>

```
popup_mode = {
  enabled      = true,
  hide_command = "tmux detach-client",
}
```

<

With this configuration, pressing `q` runs the `hide_command` instead of
closing the agenda buffer. You can bind a tmux key to show/hide the popup
session for quick agenda access.

===============================================================================
SAFETY NOTES                                                  *org-super-agenda-safety*

* Refuses edits if buffer is modified in Neovim or a swapfile is detected
* DONE items set during the session remain visible in a dedicated group
  until the agenda window is closed ("sticky DONE")
* The catch‑all "Other" group never collects DONE items

===============================================================================
HIGHLIGHTING                                              *org-super-agenda-highlight*

Each TODO state can define a color and whether to apply strike‑through. Fields
listed under a state control which parts (filename, todo, headline, priority,
date, tags) receive that highlight. See |org-super-agenda-options| → `todo_states`.
